## 7. Decisiones clave (alineadas con memoria del proyecto)

1. **Couriers ofrecidos en "Por pagar":** solo Starken y Bluexpress. **Chilexpress queda fuera** (decisión 2026-05-22).
2. **Estados de despacho:** solo Preparando, Despachado, Listo para retiro. No "Entregado" ni "En reparto".
3. **Mensaje thank-you depende del estado de pago**, no solo del envío. Antes mostraba mensaje verde aunque el pago hubiera fallado o estuviera pendiente.
4. **Email "en espera" custom solo cuando método = `bacs`** (transferencia bancaria nativa). Otros casos respetan WC nativo.
5. **Wizards bajo menú "Despachos BGMG"** como submenús (no menú top-level propio).
6. **Wizards idempotentes**: cada paso detecta su estado en vivo consultando WC, sin flags persistidos.
7. **Factura persiste en `user_meta` `_billing_bgmg_*`** para autocompletar próximo checkout (clientes empresa).
8. **Plugin asume Chile only.** Validaciones RUT/teléfono/región solo aplican si `billing_country === 'CL'`.
9. **HPOS declarado y código compatible** (incluyendo URLs de admin tras fix 1.15.2).
10. **El checkout funciona con shortcode clásico, no Block Checkout.** bgmg-landing sirve template custom vía `template_include` y dentro hace `do_shortcode('[woocommerce_checkout]')`.

## 9. Riesgos técnicos latentes (no son bugs hoy)

1. **Submenú wizards depende de orden de prioridades `admin_menu`**. `admin-despachos-menu.php` (prio 70) crea el menú padre `bgmg-despachos`. Wizards usan prio 80-82. Si alguien cambia esas prioridades, los wizards desaparecen del menú silenciosamente. Fix sugerido en Fase 10 (D1).

2. **Plugin de Transbank puede sobrescribir `woocommerce_thankyou_order_received_text`** con prioridad más alta. Si el mensaje custom no aparece tras pagar con Transbank, subir prioridad del filtro de bgmg-chile a 99.

3. **Edge case zona "Worldwide" sin location CL explícita**: el wizard de envíos no la detecta y ofrece crear duplicada. Improbable pero posible.

4. **`get_users` con `number=9999`** en wizard checkout: si >9999 clientes con RUT, no los cuenta todos. Para escala actual (estimado <500) OK.

### Comando para regenerar ZIPs (PowerShell, copy-paste):

```powershell
function New-PluginZip {
  param([string]$SourceDir, [string]$EntryPrefix, [string]$ZipPath)
  Add-Type -AssemblyName System.IO.Compression.FileSystem | Out-Null
  if (Test-Path $ZipPath) { Remove-Item $ZipPath -Force }
  $zip = [System.IO.Compression.ZipFile]::Open($ZipPath, 'Create')
  try {
    $base = (Resolve-Path $SourceDir).Path.TrimEnd('\','/')
    Get-ChildItem -Path $base -Recurse -File | ForEach-Object {
      $rel = $_.FullName.Substring($base.Length).TrimStart('\','/').Replace('\','/')
      [void][System.IO.Compression.ZipFileExtensions]::CreateEntryFromFile($zip, $_.FullName, "$EntryPrefix/$rel", [System.IO.Compression.CompressionLevel]::Optimal)
    }
  } finally { $zip.Dispose() }
}
New-PluginZip -SourceDir "C:\Users\maria\Desktop\bgmg-chile-proyecto-general\plugins\bgmg-chile" -EntryPrefix "bgmg-chile" -ZipPath "C:\Users\maria\Desktop\bgmg-chile-proyecto-general\zips\bgmg-chile.zip"
New-PluginZip -SourceDir "C:\Users\maria\Desktop\bgmg-landing" -EntryPrefix "bgmg-landing" -ZipPath "C:\Users\maria\Desktop\bgmg-chile-proyecto-general\zips\bgmg-landing.zip"
```

**Importante: NO usar `Compress-Archive` directamente** — genera ZIPs con backslashes que rompen instalación en WordPress.
